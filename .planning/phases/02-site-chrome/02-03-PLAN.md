## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| termsUrl / privacyUrl → anchor href | Merchant-supplied legal link URLs rendered as href |
| footerColumn link*Url props → anchor href | Per-block link URLs from merchant go into href |
| logo image_picker → img src via ImageGuard | Image URL from platform media library into img src |
| copyright text → JSX text node | Merchant string interpolated as JSX text |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Tampering | termsUrl / privacyUrl / footerColumn link*Url in href attributes | accept | React escapes attribute values; platform sanitizes URLs before delivery; static render — no server-side risk. Per RESEARCH.md Security Domain: javascript: URLs are low-risk for a static bundle; platform responsibility. |
| T-02-07 | Tampering | copyright, termsLabel, privacyLabel, footerColumn title/link labels rendered as text | mitigate | All text rendered via JSX text interpolation — React auto-escapes. No dangerouslySetInnerHTML. No richtext/html setting types in Footer or FooterColumn. |
| T-02-08 | Tampering | BlocksSlot opaque output from renderBlocks | accept | renderBlocks is platform-owned; output is React elements managed by the platform runtime; no raw HTML injection path. |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | Zero new package installs this plan (RESEARCH.md Package Legitimacy Audit: no new dependencies). |
</threat_model>
